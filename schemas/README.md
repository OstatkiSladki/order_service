# Схемы валидации (Schemas)

Здесь располагаются Pydantic (v2) модели. Их главная задача — строго контролировать, какие данные заходят в API (тело запроса) и какие данные могут покинуть API (тело ответа).

### Почему их две разные папки (models vs schemas)?
* `models/` общаются с **базой данных** (SQLAlchemy).
* `schemas/` общаются с интернетом, **с JSON данными** пользователя (Pydantic). 

### Логика:
* Если в модели БД есть секретный столбец `pickup_code`, мы **не добавляем** его в `OrderBase` или `Order` схему внутри `schemas/order.py`. Благодаря этому, когда FastAPI преобразует данные БД в JSON, код отсекается и хакер не может его считать.
* Для кода выделена отдельная схема `PickupCode`, которая используется только одной конкретной безопасной ручкой.
* Настроено использование `model_config = ConfigDict(from_attributes=True)` во всех схемах: это команда Pydantic "читать данные из ORM-объектов алхимии".
